	- DevOps ::::
	
		DevOps is Software Development Strategy/Process, which help to promote collaboration between the teams like Development Team and Operations Team to achieve Continuous Development, Continuous Integration, Continuous Testing, Continuous Delivery, Continuous Deployment and Continuous Monitoring in more automated fashion.

		Application Architecture :
		
			- Monolith Application Architecture:
			
				- It is tightly coupled in nature
				- All the Functions/Modules/Dependencies are tightly coupled 
				- With this only Continuous Delivery is possible. We cannot achieve Continuous Deployment
			
			- Micro-Service Based Application Architecture:
			
				- It is loosely coupled Application Architecture
				- Each Function/Module are considered as Micro-Service
				- Each Micro-Service is independently developed, testing and implement to production
				- With this we can achieve Continuous Deployment
				
	
		DevOps Stages :
		
			- Continuous Development :
				
				- It is the capabilibity of the Application Development Team to Continuously develop the changes and integrate for further testing and implementation.
				
				- It aim to improve the Developers' Productivity.
				
				Developer : 
				
				Role of any Developer ??? - Create Source Code
				
					- Create Source Code
					
					- Manual Build the Code - Compile & Create Artifacts(Binaries - *.war,*.jar,*.exec,*.dll,*.pkl)
					- Manual Unit Testing 
					- Manual Promote the Changes to Test Environment
					- Manual Notify the Testing Team - Thru Email
					
				Using DevOps Approach :::
				
					- Create Source Code
					- Save the Source Code in Source Code Repositories like github 
					
					DevOps can Automate :
					
					- Build the Code - Compile & Create Artifacts(Binaries - *.war,*.jar,*.exec,*.dll,*.pkl)
					- Unit Testing 
					- Promote the Changes to Test Environment
					- Notify the Testing Team - Thru Email

				Tools :
				
					- IDEs - Integrated Development Environment - Eclipse, Visual Studio Code, Visual Studio, Pycharm
					- Git, GitHub
							
				
			- Continuous Integration ::: CI

				- It is the capabilibity of the Application Development Team to Continuously Integrate the changes for further testing and implementation.			

					DevOps can Automate :
					
					- Build the Code - Compile & Create Artifacts(Binaries - *.war,*.jar,*.exec,*.dll,*.pkl)
					- Unit Testing 
					- Promote the Changes to Test Environment
					- Notify the Testing Team - Thru Email
					
				Tools :
					- IDEs - Integrated Development Environment - Eclipse, Visual Studio Code, Visual Studio, Pycharm
					- Git, GitHub
					- Jenkins,Build Tools(Maven),github-actions,Docker,Kubernetes,Ansible					

					
			- Continuous Testing :::
			
				- It is the capabilibity of the Testing Team to Continuously Test the changes and produce the Test Reports to allow or deny the further deployments.
				
				Tools :
				
					- TestNG,Selenium,Selenium-Grid 
					- Jenkins 
			
			
			- Continuous Delivery / Deployment

				- Continuous Delivery: 
					- Is a process to release the changes to production environments with Manual approvals.	
					- Continuous Delivery Always expects Manual Intervension/Approvals 
					- This leads to Downtime during production release

				Tools :::
				
					- Jenkins 
					- Docker, Kubernetes(Optional)
					- Ansible
					
				- Continuous Deployment:
					- Is a process to release the changes to production environments without Manual approvals.
					- Continuous Deployment Never expect any Manual Approvals.
					- Without any Downtime we can deploy the changes to production
					- ZERO Downtime.
					
					- To Achieve Continuous Deployment, we should have Micro-Service based Application Architecture.
					
				Tools :::
				
					- Jenkins 
					- Docker, Kubernetes
					- Ansible
